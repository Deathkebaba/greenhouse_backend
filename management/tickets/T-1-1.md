## Implementation Approach

### 1. HTML Markup (src/game-mount.ts)

Add two new tab buttons and sections to `GAME_VIEW_MARKUP`:
```html
<button class="drawer-tab-btn" data-tab="messages">Messages</button>
<button class="drawer-tab-btn" data-tab="relations">Relations</button>
```
```html
<section id="tab-messages" class="drawer-tab-section"></section>
<section id="tab-relations" class="drawer-tab-section"></section>
```

### 2. Table Population (src/game-ui.ts)

**Messages table** — iterate `state.messages` to produce rows with columns:
- Sender (player name via `state.playersById`)
- Receiver (player name)
- Turn (numeric)
- Message (text, potentially truncated)

Columns: `[{ label: "Sender", filterKind: "select" }, { label: "Receiver", filterKind: "select" }, { label: "Turn", filterKind: "range" }, { label: "Message", filterKind: "text" }]`

Click on sender/receiver could select that player (if a player selection mechanism exists) or focus the player entity.

**Relations table** — combine `state.relationStatuses` and `state.relationPreferences`:
- For each relation status pair, look up the corresponding preference to show a combined view.
- Columns: Player 1, Player 2, Mutual Status, My Preference, Pending.

Columns: `[{ label: "Player 1", filterKind: "select" }, { label: "Player 2", filterKind: "select" }, { label: "Status", filterKind: "select" }, { label: "My Preference", filterKind: "select" }, { label: "Pending", filterKind: "select" }]`

### 3. Filter Snapshot Extension

Extend `DrawerFilterSnapshot` type to include `messages` and `relations` filter state shapes. Update `getDrawerFilterSnapshot` to capture the new tabs.

### 4. Tests

Add test cases in `src/game-ui.test.ts` verifying:
- Messages tab renders with expected columns and row count
- Relations tab renders with expected columns and row count
- Filter/sort interactions work on the new tabs
- Tab switching includes the new tabs
