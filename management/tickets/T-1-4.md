## Implementation Approach

### 1. Type System Extensions (src/worker-query.ts)

**a) Add new query types to `DrawerQueryRequest`:**
```typescript
export type DrawerQueryRequest = BaseQueryRequest & {
  type: "drawer_locations" | "drawer_ships" | "drawer_markets"
       | "drawer_messages" | "drawer_relations";  // NEW
  filters: DrawerQueryFilters;
};
```

**b) Update `isQueryRequest` discriminator** to include the new types.

### 2. SQLite Row Types

Add row type definitions:
```typescript
type SqliteMessageRow = {
  game_id: number;
  sender: number;
  receiver: number;
  turn: number;
  message: string;
};

type SqliteRelationRow = {
  game_id: number;
  player_1: number;
  player_2: number;
  status: number;
  player_pref: number;
  target_pref: number;
  pending: number;
};
```

### 3. Required Tables Registration

Add to `QUERY_TYPE_REQUIRED_SQLITE_TABLES`:
```typescript
drawer_messages: [
  SQLITE_CACHE_SCHEMA_TABLES.messages,
  SQLITE_CACHE_SCHEMA_TABLES.players,
],
drawer_relations: [
  SQLITE_CACHE_SCHEMA_TABLES.relationStatus,
  SQLITE_CACHE_SCHEMA_TABLES.relationPreferences,
  SQLITE_CACHE_SCHEMA_TABLES.players,
],
```

### 4. Query Functions

Implement four functions following existing patterns:
- `queryDrawerMessages` (IDB fallback) — reads from IndexedDB messages store + players store
- `queryDrawerRelations` (IDB fallback) — reads from IndexedDB relation stores + players store
- `queryDrawerMessagesFromSqlite` — SQLite join of `messages` + `players`
- `queryDrawerRelationsFromSqlite` — SQLite join of `relation_status` + `relation_preferences` + `players`

**Messages drawer rows** — columns: Sender, Receiver, Turn, Message
**Relations drawer rows** — columns: Player 1, Player 2, Mutual Status, My Preference, Pending

### 5. Routing in handleQueryRequest

Add cases in `handleQueryRequest`:
```typescript
: type === "drawer_messages"
  ? await queryDrawerMessagesFromSqlite(gameId, request.filters, offset, limit)
  : await queryDrawerRelationsFromSqlite(gameId, request.filters, offset, limit)
```

### 6. UI Integration (src/game-ui.ts — connected to T-1-1)

In `populateDrawerTables`, when `queryOptions` are present, pass
`queryRequestType: "drawer_messages"` and `queryRequestType: "drawer_relations"`
to the respective `buildTable` calls so the drawer table views delegate paging
to the worker query client.

### 7. Worker Query Client Exports

Ensure `DrawerQueryRequestType` includes the new types for the UI to reference
them. The type alias `DrawerQueryRequestType` in `game-ui.ts` (line 88) must
be updated:
```typescript
type DrawerQueryRequestType = "drawer_locations" | "drawer_ships" | "drawer_markets"
                            | "drawer_messages" | "drawer_relations";
```
